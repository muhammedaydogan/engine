Generic AI Agent Container

Nihai Hedef Mimari — V1'den V3'e tam resim

Tüm yol haritası devredeyken container'ın son hâli: Cognee üçlü deposu (SQLite · LanceDB · Kuzu) dahil. Renkli oklar veri akışlarıdır; üstteki düğmelerle tek akışı vurgulayıp numaraları sırayla izleyebilirsin.

Akış
Sürüm V1 — çekirdek (bugün inşa edilen) V1.5 — yakın eklentiler V2 — Cognee + graph V3 — agent-to-agent

Okuma kuralı: etiketsiz her öğe V1'dir. Sürüm etiketi taşıyan öğe o sürümde devreye girer; V1'de yalnızca port/config kapısı bulunur. Çift uçlu oklar okuma/çağrı gidiş-dönüşünü gösterir.

Sorgu / üretim akışı

  1. 1İstemci isteği: MCP tool'u (askAgent, queryKB) veya REST (/chat, /query) — API key doğrulanır.
  2. 2API → Agent Core: system prompt kurulur (persona + runtime talimat).
  3. 3Agent Core retrieval'ı çağırır (mod config'ten: vector | hybrid | graph).
  4. 4LanceDB'de FTS + vektör arama → RRF füzyonu (ops. rerank V1.5).
  5. 5V2 Bulunan giriş node'larından Kuzu'da graph genişletme → triplet'ler.
  6. 6Bağlam, etiketli veri blokları + kaynak atıflarıyla Agent Core'a döner.
  7. 7LLM çağrısı: adapter → OpenAI-uyumlu endpoint (tool tanımları dahil).
  8. 8Model tool isterse: Tool Katmanı → MCP / harici servis; sonuç bağlama eklenir, 7'ye dönülür (≤ max_iterations).
  9. 9Cevap + atıflar istemciye; çağrı usage ledger'a (SQLite, ince noktalı ok) yazılır.

Ingest akışı (cognify)

  1. A1Kaynak girişi: build-time bake (knowledge/ + repo) ve runtime /upload; addRepo, web_fetch V1.5.
  2. A2Cognify hattı: parse → chunk (kaynak/bölüm metadata'sı). V2'de ek: LLM ile entity/ilişki çıkarımı + özetleme.
  3. A3Embedding çağrısı LLM katmanı adapter'ı üzerinden endpoint'e gider.
  4. A4Katalog + pipeline durumu → SQLite (origin: baked|runtime).
  5. A5Chunk vektörleri + FTS index → LanceDB (V2: node embedding'leri de).
  6. A6V2 Entity / ilişki / triplet'ler → Kuzu.

A2A akışı V3

  1. B1Başka bir agent container, bu container'ı sıradan bir MCP server olarak çağırır (üst kutu → /mcp).
  2. B2Bu agent'ın MCP client'ı, config'e eklenmiş diğer container'ın /mcp'sine bağlanır ve tool'unu çağırır.
  3. B3Dönen sonuç, sorgu akışının 8. adımına sıradan bir tool sonucu olarak girer — özel mekanizma yok; kapı V1 mimarisinde hazır.
